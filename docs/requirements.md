# 产品需求说明

## 1. 建设目标

建设一个人员任职资格评价 APP，用于对员工在不同能力维度上的表现进行标准化评价，并基于评价结果形成综合分数、能力等级和整体分布分析。

## 2. 角色定义

### 管理员

- 维护评价批次
- 维护评价体系
- 维护被评价人员
- 查看统计结果和个人结果
- 查看审计日志和执行数据库备份

### 评委

- 查看待评价人员
- 按评分项逐项打分
- 提交个人评价结果

### 员工

- 对本人进行自评
- 查看个人结果

### 主管

- 对人员进行主管复核评分
- 审核评审记录

## 3. 评价体系

### 3.1 等级

- 等级数量可配置
- 每个等级包含名称、排序和得分率区间
- 高等级可附加关键项门槛

### 3.2 评价维度

默认评价维度包括：

- 业务能力
- IT 技术能力
- AI 应用能力
- 通用能力

### 3.3 维度下分类

示例分类包括但不限于：

- 业务能力：工业互联网、智能制造、人工智能
- IT 技术能力：前端能力、后端能力、数据库能力、调试与问题解决能力、数据处理能力、工业互联网能力、质量优化能力、多场景应用能力、系统运维能力、系统架构能力、技术创新能力、技术战略规划能力、行业影响力、项目能力
- 通用能力：沟通表达能力、学习能力、创新能力

### 3.4 评分项

- 每个分类下可配置多个评分项
- 每个评分项包含标题、说明、权重、关键项标记
- 权重默认使用有限档位：`1.0 / 1.5 / 2.0`

## 4. 评分规则

- 单项评分档位固定为：`0 / 1 / 3`
- 单项满分 = `3 × 权重`
- 单项加权得分 = `评分值 × 权重`
- 综合得分率 = `实际加权得分总和 / 满分加权得分总和`

## 5. 定级规则

- 系统按照综合得分率自动判定等级
- 每个等级配置一个得分率区间
- 若等级启用关键项规则，则需同时满足：
  - 关键项最低得分率要求
  - 或关键项不得出现 0 分要求

## 6. 统计分析

系统对一个评价批次内所有已评分人员提供：

- 等级分布
- 维度平均得分率
- 部门分布与平均得分率
- 岗位分布与平均得分率
- 个人结果列表
- 多评价批次趋势分析

## 7. 当前版本约束

- 当前版本仍采用单体 Node.js + 原生前端实现，目标优先是业务闭环和规则验证
- 当前版本使用 SQLite 存储运行数据，并保留 JSON 作为首启导入种子
- 当前版本的 SSO 为基于账号映射的轻量接入，不包含完整企业身份协议适配
- 当前版本的权限模型以角色权限集为主，尚未细化到字段级和组织级授权
- 当前版本已支持多评委独立评分后汇总，但尚未引入复杂流程编排或外部审批系统集成
